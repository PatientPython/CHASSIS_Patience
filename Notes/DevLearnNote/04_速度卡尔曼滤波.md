# 第四章 卡尔曼滤波速度观测

## 4.1 速度观测的目的

在平衡步兵底盘系统中，准确的车身速度估计对于 LQR 控制至关重要。然而：

1. **轮速编码器**: 响应快，但易受打滑、腿部摆动影响
2. **IMU 加速度计**: 短期准确，但积分会漂移

**解决方案**: 使用卡尔曼滤波融合两种传感器：
- 轮速编码器提供**观测值**
- IMU 加速度计提供**加速度观测**
- 卡尔曼滤波输出最优的速度估计

## 4.2 卡尔曼滤波原理

### 4.2.1 状态空间模型

**状态向量** (2 维):
```
x = [v, a]ᵀ
```
| 元素 | 含义 | 单位 |
|------|------|------|
| v | 车身速度 | m/s |
| a | 车身加速度 | m/s² |

**状态方程** (匀加速模型):
```
x_k = F * x_{k-1} + w
```
其中状态转移矩阵：
```
F = [1  dt]
    [0   1]
```

**观测方程**:
```
z_k = H * x_k + v
```
本系统中 `H` 为单位矩阵，即直接观测速度和加速度。

### 4.2.2 卡尔曼滤波流程

**1. 时间更新（预测）**:
```
x_prior = F * x_{k-1}
P_prior = F * P_{k-1} * Fᵀ + Q
```

**2. 测量更新（校正）**:
```
K = P_prior * Hᵀ * (H * P_prior * Hᵀ + R)^(-1)
x = x_prior + K * (z - H * x_prior)
P = (I - K * H) * P_prior
```

其中：
- `Q`: 过程噪声协方差
- `R`: 测量噪声协方差
- `K`: 卡尔曼增益

## 4.3 卡尔曼滤波数据结构

### 4.3.1 结构体定义

```c
// API/Algorithm.h
typedef struct {
    float x[2];        // 状态向量 [vel, acc]
    float P[2][2];     // 状态估计误差协方差矩阵
    float Q[2][2];     // 过程噪声协方差矩阵
    float R[2][2];     // 测量噪声协方差矩阵
    float K[2][2];     // 卡尔曼增益矩阵
    float dt;          // 采样时间 (s)

    float VelFB;       // 滤波后的速度反馈值
    float VelNxt;      // 预测的速度值
} KF_StructTypeDef;

// 全局变量
KF_StructTypeDef GstCH_VelKF;  // 底盘速度卡尔曼滤波器
```

### 4.3.2 初始化

```c
// API/Algorithm.c L1160-1180
void KF_ChassisVel_StructInit(KF_StructTypeDef *KFptr, float dt) {
    KFptr->dt = dt;

    // 初始状态估计
    KFptr->x[0] = 0.0f;  // 速度
    KFptr->x[1] = 0.0f;  // 加速度

    // 初始协方差矩阵
    KFptr->P[0][0] = 1.0f;  KFptr->P[0][1] = 0.0f;
    KFptr->P[1][0] = 0.0f;  KFptr->P[1][1] = 1.0f;

    // 过程噪声协方差 Q
    KFptr->Q[0][0] = 0.001f;   // 速度过程噪声
    KFptr->Q[1][1] = 0.1f;     // 加速度过程噪声

    // 测量噪声协方差 R
    KFptr->R[0][0] = 0.05f;      // 轮速编码器噪声
    KFptr->R[1][1] = 0.00015f;   // IMU 加速度计噪声 (BMI088)
}
```

**参数调优指南**:
| 参数 | 增大效果 | 减小效果 |
|------|----------|----------|
| `Q[0][0]` | 速度估计更跟随观测 | 速度估计更平滑 |
| `Q[1][1]` | 加速度估计更灵敏 | 加速度估计更稳定 |
| `R[0][0]` | 降低轮速权重 | 增加轮速权重 |
| `R[1][1]` | 降低 IMU 权重 | 增加 IMU 权重 |

## 4.4 卡尔曼滤波 API 函数

### 4.4.1 时间更新（预测）

```c
// API/Algorithm.c L1187-1210
void KF_ChassisVel_Predict(KF_StructTypeDef *KFptr) {
    // 1. 先验状态估计 x_prior = F * x_{k-1}
    float v_old = KFptr->x[0];
    float a_old = KFptr->x[1];

    // x_k_prior: v = v + a*dt, a = a
    KFptr->x[0] = v_old + a_old * KFptr->dt;
    KFptr->x[1] = a_old;

    // 2. 先验协方差矩阵 P_prior = F * P * Fᵀ + Q
    float p00 = KFptr->P[0][0], p01 = KFptr->P[0][1];
    float p10 = KFptr->P[1][0], p11 = KFptr->P[1][1];
    float dt = KFptr->dt;

    // F * P * Fᵀ 展开计算（F = [[1,dt],[0,1]]）
    KFptr->P[0][0] = p00 + dt*(p10 + p01) + dt*dt*p11 + KFptr->Q[0][0];
    KFptr->P[0][1] = p01 + dt*p11 + KFptr->Q[0][1];
    KFptr->P[1][0] = p10 + dt*p11 + KFptr->Q[1][0];
    KFptr->P[1][1] = p11 + KFptr->Q[1][1];
}
```

**物理含义**:
- 速度预测：`v_pred = v + a * dt`（匀加速运动学）
- 加速度预测：`a_pred = a`（保持不变）
- 协方差增长：随时间推移，预测不确定性增加

### 4.4.2 测量更新（校正）

```c
// API/Algorithm.c L1222-1279
void KF_ChassisVel_Update(KF_StructTypeDef *KFptr,
                          float v_body_obs, float a_imu) {
    // 1. 计算卡尔曼增益 K
    // S = H * P_prior * Hᵀ + R = P + R (H 为单位阵)
    float S00 = KFptr->P[0][0] + KFptr->R[0][0];
    float S01 = KFptr->P[0][1] + KFptr->R[0][1];
    float S10 = KFptr->P[1][0] + KFptr->R[1][0];
    float S11 = KFptr->P[1][1] + KFptr->R[1][1];

    // 计算 S 的逆矩阵（2x2 矩阵求逆公式）
    float det = S00*S11 - S01*S10;
    if (fabsf(det) < 1e-12f) return;  // 防止矩阵奇异

    float invS00 =  S11 / det;
    float invS01 = -S01 / det;
    float invS10 = -S10 / det;
    float invS11 =  S00 / det;

    // K = P_prior * Hᵀ * invS = P * invS
    KFptr->K[0][0] = KFptr->P[0][0]*invS00 + KFptr->P[0][1]*invS10;
    KFptr->K[0][1] = KFptr->P[0][0]*invS01 + KFptr->P[0][1]*invS11;
    KFptr->K[1][0] = KFptr->P[1][0]*invS00 + KFptr->P[1][1]*invS10;
    KFptr->K[1][1] = KFptr->P[1][0]*invS01 + KFptr->P[1][1]*invS11;

    // 2. 后验状态估计 x = x_prior + K * (z - H * x_prior)
    float z0 = v_body_obs;  // 速度观测值
    float z1 = a_imu;       // 加速度观测值

    float y0 = z0 - KFptr->x[0];  // 速度残差
    float y1 = z1 - KFptr->x[1];  // 加速度残差

    KFptr->x[0] += KFptr->K[0][0]*y0 + KFptr->K[0][1]*y1;
    KFptr->x[1] += KFptr->K[1][0]*y0 + KFptr->K[1][1]*y1;

    // 3. 后验协方差矩阵 P = (I - K) * P_prior
    float I_K00 = 1.0f - KFptr->K[0][0];
    float I_K01 =        - KFptr->K[0][1];
    float I_K10 =        - KFptr->K[1][0];
    float I_K11 = 1.0f - KFptr->K[1][1];

    float P_new[2][2];
    P_new[0][0] = I_K00*KFptr->P[0][0] + I_K01*KFptr->P[1][0];
    P_new[0][1] = I_K00*KFptr->P[0][1] + I_K01*KFptr->P[1][1];
    P_new[1][0] = I_K10*KFptr->P[0][0] + I_K11*KFptr->P[1][0];
    P_new[1][1] = I_K10*KFptr->P[0][1] + I_K11*KFptr->P[1][1];

    memcpy(KFptr->P, P_new, sizeof(P_new));
}
```

**计算说明**:
1. **S 矩阵**: 新息协方差，反映观测值的不确定性
2. **K 矩阵**: 卡尔曼增益，决定相信预测还是观测
   - `K → 0`: 完全相信预测
   - `K → 1`: 完全相信观测
3. **残差修正**: `y = z - x_prior` 用残差修正状态估计

## 4.5 速度融合主处理函数

### 4.5.1 观测值计算

**关键问题**: 轮速编码器测量的是轮子相对于车身的速度，而非车身相对于地面的速度。

**解决方案**: 扣除腿部摆动引起的轮速分量：

```c
// Application/Chassis/Chassis_APIFunction.c L485-516
void CH_VelKF_Process(void) {
    // 1. 获取测量数据
    // 轮子测量的线速度（左右轮平均）
    float v_wheel_theory = (GSTCH_HM1.AngleVelFB + GSTCH_HM2.AngleVelFB) / 2.0f * R_w;

    // IMU 测量的机体加速度
    float a_imu = GSTCH_Data.AccXFB;

    // 腿部摆动引起的轮速（运动学补偿）
    float v_rel_leg = (GSTCH_Data.xC1_dot + GSTCH_Data.xC2_dot) / 2.0f;

    // 重要：观测到的底盘速度 = 轮速 - 腿部摆动速度
    // 物理意义：车身相对于地面的速度
    float v_body_obs = v_wheel_theory - v_rel_leg;

    // 2. 离地处理（动态调节 R 矩阵）
    if (GSTCH_Data.F_OffGround1 || GSTCH_Data.F_OffGround2) {
        // 离地时轮速不可信，大幅增加 R[0][0]
        GstCH_VelKF.R[0][0] = 1000000.0f;  // 几乎不使用轮速观测
    } else {
        // 正常状态
        GstCH_VelKF.R[0][0] = 0.05f;
    }

    // 3. 执行速度预测
    KF_ChassisVel_Predict(&GstCH_VelKF);
    GstCH_VelKF.VelNxt = GstCH_VelKF.x[0];

    // 4. 执行速度更新
    KF_ChassisVel_Update(&GstCH_VelKF, v_body_obs, a_imu);
    GstCH_VelKF.VelFB = GstCH_VelKF.x[0];

    // 5. 更新全局变量
    GSTCH_Data.VelFB = GstCH_VelKF.VelFB;  // 融合后的速度
    GSTCH_Data.DisFB += GSTCH_Data.VelFB * GCH_TaskTime;  // 位移积分

    GSTCH_Data.VelBody_HM_Obs = v_body_obs;  // 保存观测值用于调试
}
```

### 4.5.2 离地处理的物理意义

**离地时轮速不可信的原因**:
- 轮子悬空，编码器测量的是轮子相对于车身的转速
- 轮速不再反映车身速度

**解决方案**:
- 增大 `R[0][0]` → 卡尔曼增益 `K[0][0]` 减小 → 降低轮速观测权重
- KF 主要依赖 IMU 加速度积分预测速度

## 4.6 API 调用示例

```c
// 在 ChassisTask 1kHz 循环中

// 1. 执行速度融合
CH_VelKF_Process();

// 2. 读取融合后的速度
float vel = GSTCH_Data.VelFB;  // 融合速度 (m/s)

// 3. 读取预测速度（用于前馈）
float vel_pred = GstCH_VelKF.VelNxt;

// 4. 读取卡尔曼增益（用于调试）
float K_vel = GstCH_VelKF.K[0][0];  // 速度观测增益
if (K_vel < 0.1f) {
    // KF 更相信预测（可能离地了）
}
```

## 4.7 调试建议

### 4.7.1 观测变量

| 变量 | 含义 | 正常范围 |
|------|------|----------|
| `GstCH_VelKF.x[0]` | 融合速度 | -2~2 m/s |
| `GstCH_VelKF.x[1]` | 融合加速度 | -5~5 m/s² |
| `GstCH_VelKF.K[0][0]` | 速度卡尔曼增益 | 0.3~0.8 |
| `GSTCH_Data.VelBody_HM_Obs` | 速度观测值 | -2~2 m/s |
| `GSTCH_Data.VelFB` | 最终融合速度 | -2~2 m/s |

### 4.7.2 传感器对比

建议同时观测以下变量，验证融合效果：

```
1. 纯轮速：v_wheel = (HM1_AngleVel + HM2_AngleVel) / 2 * R_w
2. 纯 IMU 积分：v_imu = ∫ a_imu dt
3. 融合速度：v_kf = GSTCH_Data.VelFB
```

**正常现象**:
- 静止时：三种速度都应接近 0
- 匀速时：轮速和融合速度接近，IMU 积分可能有漂移
- 加速时：融合速度介于两者之间，响应比 IMU 快，噪声比轮速小

### 4.7.3 参数整定流程

1. **初始值**: 使用代码中的默认参数
2. **观察轮速噪声**: 静止时轮速的标准差 → 调整 `R[0][0]`
3. **观察 IMU 噪声**: 静止时 IMU 加速度的标准差 → 调整 `R[1][1]`
4. **验证离地响应**: 抬腿时观察速度跳变 → 调整离地时的 `R[0][0]`

## 4.8 常见问题

### 4.8.1 速度估计漂移

**现象**: 静止时速度估计缓慢增加

**原因**: IMU 零偏导致加速度积分漂移

**解决**:
1. 减小 `Q[1][1]`（降低加速度过程噪声）
2. 增加 `R[1][1]`（降低 IMU 权重）
3. 检查 IMU 零偏校准

### 4.8.2 速度估计噪声大

**现象**: 速度估计有明显抖动

**原因**: 轮速噪声过大或卡尔曼增益过大

**解决**:
1. 增加 `R[0][0]`（降低轮速权重）
2. 增加 `Q[0][0]`（增加过程噪声，使 K 减小）
3. 检查轮速编码器接线
